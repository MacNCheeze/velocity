Velocity Class

Defined in: core.js:12
Module: Velocity

The Velocity object provides a common API for test frameworks to report test results. Frameworks can also request assets, such as a copy of the user's application (the 'mirror') in which tests can be safely run without impacting on-going development.

Test results and log activity are reported via Meteor methods.

(
  • name
  • [options]
)

Defined in core.js:150

Registers a testing framework plugin.

Parameters:

  • name String

    The name of the testing framework.

  • [options] Object optional

    Options for the testing framework.

    • disableAutoReset String

      Velocity's reset cycle will skip reports and logs for this framework It will be the responsibility of the framework to clean up its **!

    • regex String

      The regular expression for test files that should be assigned to the testing framework. The path relative to the tests folder is matched against it. The default is "name/.+.js$" (name is the testing framework name).

    • sampleTestGenerator Function

      sampleTestGenerator returns an array of fileObjects with the following fields:

      • path String
        relative path to place test file (from PROJECT/tests)
      • contents String
        contents of the test file the path that's returned
